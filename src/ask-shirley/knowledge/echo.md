# Echo

## One-sentence description
Echo is a generative poetry studio where pasted language continues breathing across five afterlives — one analysis spine, multiple renderers.

## The original motivation
Public case-study copy frames Echo as a response to modern communication compressing meaning into fragments: words are meant to carry emotion, memory, identity, and intention, yet speed often strips emotional weight. Echo treats a manuscript as material for a field rather than a screenshot of a word cloud or a chat that forgets the page.

## The problem being explored
How to keep written language moving — cluster, drift, orbit, residue — so relationships among words can settle into feeling instead of being summarized as a static caption.

## Shirley's role
Designer & Developer on a three-person CS146J (Stanford) team with Clare Lei and Rita Xiang (2024–2025).

## Important technical decisions
- One shared analysis pipeline feeds every form.
- Local tokenization and co-occurrence can run in the browser when the API is unreachable; server-side Datamuse can extend the field with semantic neighbors.
- Shared controls (intensity, density, motion) keep five renderers on the same spine.
- Forms named in the portfolio: Constellation (Network), Drift (Soup), Imprint (ASCII), Spiral (Vortex), Gravity (Orbit).
- Network / Constellation uses D3 force layout over co-occurrence; it is described as ancestor of MuseLab's Pulse.

## Important visual or experiential decisions
- Moonlight and Paper themes only — monochrome on purpose; chrome stays quiet so the field can speak.
- Sound is part of the system (low studio loop), not decoration.
- Night is the default; paper exists for reading, not product-dashboard aesthetics.
- Portfolio room uses Echo's own "Moonlight" dialect rather than a generic case-study template.
