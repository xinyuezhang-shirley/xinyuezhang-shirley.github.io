# MuseLab

## One-sentence description
MuseLab is a literary workshop dossier — interpretation over generation — where a manuscript yields structured critique instead of chat-bubble praise.

## The original motivation
Public case-study copy: Shirley wanted better feedback on her own poetry. Most AI writing tools offered vague praise, generic craft advice, and a chat box treating a poem like a support ticket — counting repetitions without saying what the draft was about.

## The problem being explored
How to engineer literary interpretation as layered argument (evidence → pattern → claim → revision question) without pretending the system is a second human critic or rewriting the writer's voice.

## Shirley's role
Designer & Developer · Independent project · 2025–2026.

## Important technical decisions
- Hybrid grounding before opinion: deterministic poetic features and optional classifiers run first as sensors, not judges.
- Specialist agents read in parallel; synthesis reconciles without averaging disagreement into mush.
- Folio structure (tabs such as Draft, Pulse, Margins, Paths, References, Revision, Imprint) instead of a single chat reply as the workshop.
- Pulse force field is supporting evidence (lemmas linked by honest co-presence), with structural reading naming organizing concept, clusters, and absences.
- Voice preservation is explicit — no rewrite of the writer's poem into the model's voice.
- Tools listed in portfolio metadata: React, TypeScript.

## Important visual or experiential decisions
- Charts and graphs are evidence; the reading is the product.
- Selecting a word on Draft traces across Pulse and Margins so commentary stays tethered to language.
- Imprint closes with residue rather than a CTA / engagement bait.
- Interface stays calm while analysis runs; each sheet has one job.

## What Shirley learned
Documented: interpretation can be engineered as layered argument without pretending to be a second human critic; keep poetry agents and product-steward agents separate; document taste so the interface cannot quietly become a SaaS template.
